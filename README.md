# MKK Proving Ground

Модульный полигон для Arma 3 с техническим именем `mkk_ptg` и префиксом сборки `ptg`. Проект предназначен для быстрой проверки техники, вооружения, боеприпасов, траекторий, пробития, телепортации и локальных утилит игрока.

## Состав модулей

- `ptg_main` — базовая конфигурация, горячие клавиши, открытие UI, виртуальный/ACE арсенал, копирование classname, удаление и разблокировка объекта под прицелом.
- `ptg_common` — общие helpers, безопасное чтение конфигов, локализация, масштабирование UI/HUD.
- `ptg_catalog` — динамический каталог `CfgVehicles`, фильтрация, сортировка, определение типа техники и совместимых ящиков БК для статики.
- `ptg_ui` — главное окно, стартовое меню, экран создания техники, камера, телепорт, перевооружение, тестовые цели, настройки HUD и обработчики кнопок.
- `ptg_spawn` — серверный спавн техники и тестовых целей, регистрация созданных объектов, удаление и очистка полигона.
- `ptg_tracking` — слежение за projectile, HUD tracking, линии траектории и маркеры снарядов на карте.
- `ptg_penetration` — тест пробития техники, orbit-камера, тестовый projectile, отчет по damage/hitpoints/crew и параметры `CfgAmmo`.
- `ptg_player` — бесконечные патроны, режим бога, обработчики выстрелов и респавна.
- `ptg_player_ace` — совместимость режима бога с ACE medical damage.
- `ptg_ace` — ACE self-action и ACE action на терминалах.

## Зависимости

- Arma 3 версии не ниже 2.20.
- CBA_A3.
- ACE3 для модулей `ptg_ace` и `ptg_player_ace`; `ptg_player_ace` помечен как optional через `skipWhenMissingDependencies`.

## Текущий функционал

- чистое стартовое меню вместо открытия каталога сразу;
- динамический каталог техники из `CfgVehicles` без жесткой привязки к модовым фракциям;
- фильтры по стороне, фракции, типу техники и поиску по названию/classname;
- карточка выбранной техники с изображением, фракцией, типом, группой, экипажем и источником мода;
- серверный спавн пустой техники или техники с экипажем;
- совместимые ящики БК для статического вооружения;
- отдельное меню тестовых целей и серверное создание/удаление целей;
- телепорт по клику на карте;
- свободная камера: выбор области на карте, создание камеры в выбранной точке, управление WASD/Q/Z/мышью/колесом, перенос камеры по ЛКМ и закрытие через настраиваемую клавишу;
- перевооружение техники игрока: выбор позиции экипажа/турели, выбор оружия, загрузка совместимых магазинов, очистка оружия и работа с пилонами через `setPylonLoadout`;
- tracking projectile с локальной камерой, HUD, таймаутом и cooldown;
- линии траектории с настройкой цвета и толщины;
- маркеры снарядов на карте с опциональной подписью ammo classname;
- статус-дисплей объектов при наведении и настраиваемые поля damage/hitpoints;
- инспектор hitpoints для техники под прицелом;
- тест пробития с выбором техники и боеприпаса, orbit-камерой, выстрелом по выбранной точке, отчетом повреждений и копированием ammo classname;
- бесконечные патроны и режим бога с повторным применением после респавна;
- виртуальный арсенал и ACE3 Arsenal через горячие клавиши;
- копирование classname объекта под прицелом;
- удаление объекта под прицелом через серверный поток;
- разблокировка техники под прицелом;
- масштаб интерфейса: small, normal, large, extra large;
- локализация видимого текста через `addons/main/stringtable.xml` на английский и русский языки.

## Что проверять в игре

1. Аддон загружается без ошибок зависимостей и `CfgPatches`.
2. Горячая клавиша открытия интерфейса и ACE actions открывают полигон.
3. Стартовое меню показывает актуальные функции и не открывает каталог до выбора “Создание техники”.
4. Каталог собирается, фильтруется и показывает корректную карточку техники.
5. Спавн техники, спавн с экипажем, совместимые ящики БК для статики и очистка работают на dedicated/серверный поток.
6. Меню тестовых целей создает и удаляет цели через сервер.
7. Телепорт по карте переносит `vehicle player` в выбранную точку.
8. Камера запускается из выбранной области карты, управляется локально, переносится ЛКМ и корректно закрывается горячей клавишей.
9. Перевооружение работает для турелей, магазинов и пилонов Arma 3.
10. Tracking, линии траектории и маркеры карты запускаются только локально и корректно очищают runtime-состояние.
11. Статус-дисплей и инспектор hitpoints скрываются при открытой карте, главном окне и окне теста пробития.
12. Тест пробития создает цель, запускает orbit-камеру, стреляет в выбранную точку и обновляет отчет.
13. Бесконечные патроны, режим бога и ACE medical блокировка сохраняют ожидаемое поведение после респавна.
14. Все пользовательские строки берутся из stringtable и имеют English/Russian значения.

## Документация

Основные актуальные документы находятся в `docs/ptg/`:

- `overview.md` — идея продукта, стиль и принципы;
- `ui-ux.md` — структура интерфейса, стартовое меню, экран техники, камера, перевооружение и оверлейs;
- `catalog-and-spawn.md` — каталог, фильтры, спавн, ящики БК и очистка;
- `features.md` — телепорт, камера, tracking, траектории, маркеры, hitpoints, тест пробития, доступ и настройки;
- `architecture-and-локализацию.md` — модули, зависимости, UI-поверхности, серверные потоки и правила локализации;
- `design-full.md` — полный исходный дизайн, использовать только если раздельных документов недостаточно.

README модулей в `addons/*/README.md` являются краткими заметками; при расхождениях приоритет имеют `docs/ptg/*.md` и текущий код.
